# CLI verbs follow declarative-reconcile idiom: `apply` and `destroy`, not `install`/`uninstall`

The two lifecycle verbs are `acron apply` (reconcile OS units to the Config: create + update + prune) and `acron destroy` (remove all acron-owned units from this machine, leaving the Config intact). We deliberately avoid `install`/`uninstall`.

`install` reads as add-only and undersells the auto-prune that `apply` performs, and `uninstall` implies a paired `install` that does not exist in our vocabulary. We use the dominant on-demand declarative-reconcile pair `apply`/`destroy` (Terraform, Pulumi; `apply` also matches chezmoi, acron's closest analog: per-user, multi-machine, config-to-local-artifacts). Terraform's `apply` already removes resources absent from config, so it carries the auto-prune meaning. We rejected `sync` because its strongest pedigree (ArgoCD/Flux) is a continuous server-side controller loop, not an on-demand CLI, and pairing it with `destroy` mixes idioms. We rejected `clean` because in established tools (`git clean`, `brew bundle cleanup`) it means "prune what is not in the manifest", which is what `apply` already does, and `purge` (apt, chezmoi) because it connotes removing the tool's own config too, which `destroy` must not do.
